# 第三课：MMA 指令详解

## 3.1 MMA 指令完整格式

### 标准格式分解

```
mma.sync.aligned.m<M>n<N>k<K>.<Dtype>.<Atype>.<Btype>.<Ctype>.<layout>
    {d0, d1, ...},     // 输出寄存器 (D 矩阵)
    {a0, a1, ...},     // A 矩阵寄存器
    {b0, b1, ...},     // B 矩阵寄存器
    {c0, c1, ...};     // 累加器寄存器 (C 矩阵)
```

### 字段详细解析

```
┌──────────────────────────────────────────────────────────────────┐
│  字段名         │ 含义               │ 示例值                     │
├──────────────────────────────────────────────────────────────────┤
│  mma           │ 指令类型           │ mma (Matrix Multiply)      │
│  .sync         │ 同步模式           │ .sync (同步)               │
│  .aligned      │ 对齐模式           │ .aligned (对齐访问)        │
│  .m<M>         │ M 维度大小         │ .m16 (M=16)                │
│  .n<N>         │ N 维度大小         │ .n8 (N=8)                  │
│  .k<K>         │ K 维度大小         │ .k8 (K=8)                  │
│  .<layout>     │ 布局模式           │ .row.col (A 行优先，B 列优先)│
│  .<Dtype>      │ 输出数据类型       │ .f32 (FP32)                │
│  .<Atype>      │ A 矩阵数据类型     │ .f16 (FP16)                │
│  .<Btype>      │ B 矩阵数据类型     │ .f16 (FP16)                │
│  .<Ctype>      │ 累加器数据类型     │ .f32 (FP32)                │
└──────────────────────────────────────────────────────────────────┘
```

## 3.2 同步模式详解

### .sync vs 异步

```cuda
// 同步模式 ( blocking)
mma.sync.aligned.m16n8k8...
// 指令执行完成后才继续
// 保证数据一致性
// 适用于通用场景

// 异步模式 (non-blocking, SM90+)
mma.async.aligned.m64n8k32...
// 发起后立即返回
// 需要手动等待完成
// 适用于流水线优化
```

### 同步模式选择

```cuda
// 场景 1: 简单 MMA，直接用结果
mma.sync.aligned.m16n8k8... {d}, {a}, {b}, {c};
float result = d[0];  // 安全，已同步

// 场景 2: 异步流水线 (SM90+)
mma.async.aligned.m64n8k32... {d}, {a}, {b}, {c};
// 继续其他工作...
fence.async;
mbarrier.wait;  // 显式等待
```

## 3.3 对齐模式

### .aligned vs .unaligned

```cuda
// 对齐访问 (推荐)
mma.sync.aligned.m16n8k8...
// 硬件假设：寄存器边界对齐
// 性能最优
// 需要确保数据布局正确

// 非对齐访问 (fallback)
mma.sync.unaligned.m16n8k8...
// 处理任意对齐
// 性能略低
// 自动处理边界情况
```

### 对齐要求

```
寄存器对齐要求:

FP16 MMA (m16n8k8):
┌────────────────────────────────────────┐
│  A 寄存器：32-bit 边界 (4 FP16/寄存器) │
│  B 寄存器：32-bit 边界 (2 FP16/寄存器) │
│  C/D 寄存器：32-bit 边界 (4 FP16/寄存器)│
└────────────────────────────────────────┘

FP32 MMA (m16n8k8):
┌────────────────────────────────────────┐
│  A 寄存器：32-bit 边界 (1 FP32/寄存器) │
│  B 寄存器：32-bit 边界 (1 FP32/寄存器) │
│  C/D 寄存器：32-bit 边界 (1 FP32/寄存器)│
└────────────────────────────────────────┘
```

## 3.5 数据类型详解

### FP16 (半精度浮点)

```cuda
// FP16 GEMM (最常用)
mma.sync.aligned.m16n8k8.row.col.f32.f16.f16.f32
    {d0, d1, d2, d3},  // 4×FP32 = 16×FP16 输出
    {a0, a1},          // 2×FP32 = 8×FP16 (16×8 A 矩阵)
    {b0},              // 1×FP32 = 4×FP16 (8×8 B 矩阵)
    {c0, c1, c2, c3};  // 4×FP32 = 16×FP16 累加器

// FP16 寄存器布局:
// A 矩阵 (16×8):
//   a0 = {A[0:1, 0], A[0:1, 1], A[0:1, 2], A[0:1, 3]}
//   a1 = {A[0:1, 4], A[0:1, 5], A[0:1, 6], A[0:1, 7]}
// B 矩阵 (8×8):
//   b0 = {B[0:3, 0:1]}  // 特殊布局
// C/D 矩阵 (16×8):
//   d0 = {D[0:1, 0:3]}
//   d1 = {D[0:1, 4:7]}
//   d2 = {D[8:9, 0:3]}
//   d3 = {D[8:9, 4:7]}
```

### BF16 (Bfloat16)

```cuda
// BF16 GEMM (AI 训练推荐)
mma.sync.aligned.m16n8k16.row.col.f32.bf16.bf16.f32
    {d0, d1, d2, d3},  // 4×FP32
    {a0, a1},          // 2×FP32 = 8×BF16
    {b0, b1},          // 2×FP32 = 8×BF16
    {c0, c1, c2, c3};  // 4×FP32

// BF16 vs FP16:
// - 相同：10 位尾数
// - 不同：BF16 有 8 位指数 (同 FP32)
// - 优势：范围大，训练稳定
```

### TF32 (TensorFloat32)

```cuda
// TF32 GEMM (FP32 精度加速)
mma.sync.aligned.m16n8k8.row.col.f32.tf32.tf32.f32
    {d0, d1, d2, d3},  // 4×FP32
    {a0, a1},          // 2×TF32 (每个 TF32=19-bit)
    {b0, b1},          // 2×TF32
    {c0, c1, c2, c3};  // 4×FP32

// TF32 特点:
// - 8 位指数 (同 FP32)
// - 10 位尾数 (同 FP16)
// - 范围同 FP32，精度~3 位十进制
// - 无需修改代码，自动加速 FP32 GEMM
```

### FP8 (8 位浮点)

```cuda
// FP8 E4M3 GEMM
mma.sync.aligned.m64n8k32.row.col.f32.f8e4m3.f8e4m3.f32
    {d0, d1, d2, d3, d4, d5, d6, d7},  // 8×FP32
    {a0, a1, a2, a3, a4, a5, a6, a7},  // 8×FP8
    {b0, b1, b2, b3},                   // 4×FP8
    {c0, c1, c2, c3, c4, c5, c6, c7};  // 8×FP32

// FP8 E5M2 GEMM
mma.sync.aligned.m64n8k32.row.col.f32.f8e5m2.f8e5m2.f32
    {...}, {...}, {...}, {...};

// 注意:FP8 需要 SM89+ (Ada) 或 SM90+ (Hopper)
```

### FP64 (双精度)

```cuda
// FP64 GEMM (科学计算)
mma.sync.aligned.m8n8k4.row.col.f64.f64.f64.f64
    {d0, d1},        // 2×FP64
    {a0, a1},        // 2×FP64
    {b0, b1},        // 2×FP64
    {c0, c1};        // 2×FP64

// 注意：FP64 需要 SM80+ (Ampere+)
```

## 3.6 布局模式详解

### .row.col (最常用)

```cuda
// A 是 row-major (行优先)
// B 是 col-major (列优先)
mma.sync.aligned.m16n8k8.row.col.f16.f16.f16.f16
    {d}, {a}, {b}, {c};

内存布局:
A (16×8, row-major):
┌────────────────────────────────────────┐
│ A00 A01 A02 A03 A04 A05 A06 A07       │  ← 行 0
│ A10 A11 A12 A13 A14 A15 A16 A17       │  ← 行 1
│ ...                                    │
│ A15,0 ... A15,7                        │  ← 行 15
└────────────────────────────────────────┘

B (8×8, col-major):
┌────────────────────────────────────────┐
│ B00 B01 B02 B03 B04 B05 B06 B07       │  ← 列 0-7
│ B10 B11 B12 B13 B14 B15 B16 B17       │
│ ...                                    │
│ B70 B71 B72 B73 B74 B75 B76 B77       │
└────────────────────────────────────────┘
```

### .row.row

```cuda
// A 和 B 都是 row-major
mma.sync.aligned.m16n8k8.row.row.f16.f16.f16.f16
    {d}, {a}, {b}, {c};

// 适用场景：B 矩阵已转置
```

### .col.col

```cuda
// A 和 B 都是 col-major
mma.sync.aligned.m16n8k8.col.col.f16.f16.f16.f16
    {d}, {a}, {b}, {c};

// 适用场景：A 矩阵已转置
```

### .col.row

```cuda
// A 是 col-major, B 是 row-major
mma.sync.aligned.m16n8k8.col.row.f16.f16.f16.f16
    {d}, {a}, {b}, {c};

// 适用场景：A 和 B 都已转置
```

## 3.9 寄存器布局详解

### 16×8×8 FP16 MMA 寄存器映射

```
32 个线程 (Warp) 参与 1 个 16×8×8 MMA:

输出 D (16×8 FP16):
┌─────────────────────────────────────────────────────────┐
│  Thread 0: D[0:1, 0:1]  Thread 1: D[0:1, 2:3]  ...     │
│  Thread 2: D[2:3, 0:1]  Thread 3: D[2:3, 2:3]  ...     │
│  ...                                                     │
│  Thread 30: D[14:15, 6:7]  Thread 31: D[14:15, 8:9]    │
└─────────────────────────────────────────────────────────┘

每个线程的输出寄存器:
d0 = {D[0, 0], D[0, 1], D[1, 0], D[1, 1]}  // 4×FP16 = 32-bit

A 矩阵寄存器 (每线程):
a0 = {A[0, 0], A[0, 1], A[1, 0], A[1, 1]}  // 4×FP16 = 32-bit
a1 = {A[0, 2], A[0, 3], A[1, 2], A[1, 3]}  // 4×FP16 = 32-bit

B 矩阵寄存器 (每线程):
b0 = {B[0, 0], B[0, 1], B[1, 0], B[1, 1]}  // 4×FP16 = 32-bit
```

### 线程到 MMA 的映射

```
Warp (32 线程) → MMA 映射:

对于 m16n8k8:
- 32 线程 × 4 FP16/线程 = 128 FP16 (完整输出)
- 线程布局：(4 线程/MMA 行) × (8 线程/MMA 列)

对于 m16n8k16:
- 32 线程 × 4 FP16/线程 = 128 FP16
- 但 K 翻倍，需要更多 A/B 寄存器

对于 m16n8k32:
- 32 线程 × 4 FP16/线程 = 128 FP16
- K=32，需要 4 次迭代完成
```
