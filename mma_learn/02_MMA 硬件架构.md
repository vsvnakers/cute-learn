## 2.2 Volta (SM70) - 第一代 Tensor Core

### 架构特点

```
┌─────────────────────────────────────────────────────────┐
│                    Volta SM70 架构                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  CUDA Core 布局：                                        │
│  ┌─────────────────────────────────────────────┐       │
│  │  64 个 FP32 Core                             │       │
│  │  64 个 INT32 Core                            │       │
│  │  8 个 Tensor Core                            │       │
│  │  4 个纹理单元                                │       │
│  └─────────────────────────────────────────────┘       │
│                                                         │
│  Tensor Core 规格：                                      │
│  - 每个 SM: 8 个 TC                                      │
│  - 每个 TC: 4 个 FP16 乘加单元/cycle                     │
│  - 单 cycle 吞吐：8×4 = 32 FP16 操作                    │
│  - 实际吞吐：4×4×4 = 64 FMA/cycle (等效 1024 FMA)       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### MMA 指令集

```cuda
// Volta 支持的 MMA 形状
m8n8k4   // 8×8×4 矩阵乘法
m16n8k8  // 16×8×8 矩阵乘法

// 支持的数据类型
.f16     // FP16 (半精度)
```

### Volta MMA 吞吐量计算

```
V100 规格:
- SM 数量：80 个
- 每个 SM 的 TC 数量：8 个
- 每个 TC 的吞吐：64 FMA/cycle (4×4×4)
- 时钟频率：1455 MHz

FP16 吞吐计算:
80 SM × 8 TC/SM × 64 FMA/cycle × 1.455 GHz × 2 (FMA = 2 OP)
= 80 × 8 × 64 × 1.455 × 10^9 × 2
≈ 119,500 GFLOPS
≈ 120 TFLOPS (稠密)

考虑稀疏化 (2:4):
≈ 240 TFLOPS (稀疏)
```

### Volta MMA 限制

```cpp
// ❌ Volta 不支持的操作
mma.sync.aligned.m16n8k16...  // K=16 不支持
mma.sync.aligned.m16n8k32...  // K=32 不支持
mma.sync...f32...             // FP32 输出不支持
mma.sync...bf16...            // BF16 不支持
mma.sync...tf32...            // TF32 不支持

// ✅ Volta 支持的操作
mma.sync.aligned.m16n8k8.row.col.f16.f16.f16.f16  // OK
mma.sync.aligned.m8n8k4.row.col.f16.f16.f16.f16   // OK
```

## 2.3 Turing (SM75) - 第二代 Tensor Core

### 新增特性

```
┌─────────────────────────────────────────────────────────┐
│                   Turing SM75 新增特性                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 新增 MMA 形状：                                      │
│     - m8n8k4 (保留)                                     │
│     - m16n8k8 (保留)                                    │
│     - m16n8k16 (新增！)                                 │
│                                                         │
│  2. 新增数据类型：                                       │
│     - INT8 (推理加速)                                   │
│     - INT4 (更低精度推理)                               │
│     - FP16 Accumulator (可选 FP32)                      │
│                                                         │
│  3. 稀疏化支持：                                         │
│     - 2:4 结构化稀疏                                    │
│     - 性能翻倍                                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Turing MMA 指令

```cuda
// INT8 GEMM (4×4 吞吐提升)
mma.sync.aligned.m8n8k4.row.col.s32.s8.s8.s32
    {d0, d1}, {a0}, {b0}, {c0, c1};

// INT4 GEMM (8×8 吞吐提升)
mma.sync.aligned.m8n8k4.row.col.s32.s4.s4.s32
    {d0, d1}, {a0}, {b0}, {c0, c1};

// FP16 with FP32 Accumulator (更高精度)
mma.sync.aligned.m16n8k8.row.col.f32.f16.f16.f32
    {d0, d1, d2, d3}, {a0, a1}, {b0}, {c0, d1, d2, d3};
```

### Turing 性能提升

```
RTX 2080 Ti 规格:
- SM 数量：68 个
- 每个 SM 的 TC 数量：8 个
- 时钟频率：1635 MHz

FP16 吞吐 (稠密):
68 × 8 × 64 × 1.635 × 2 ≈ 114 TFLOPS

FP16 吞吐 (2:4 稀疏):
≈ 228 TFLOPS

INT8 吞吐 (推理):
68 × 8 × 128 × 1.635 ≈ 114 TOPS (稠密)
≈ 228 TOPS (稀疏)
```

## 2.4 Ampere (SM80) - 第三代 Tensor Core

### 重大升级

```
┌─────────────────────────────────────────────────────────┐
│                  Ampere SM80 重大升级                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 新增 MMA 形状：                                      │
│     - m16n8k8  (保留)                                   │
│     - m16n8k16 (保留)                                   │
│     - m16n8k32 (新增！K 翻倍)                           │
│                                                         │
│  2. 新增数据类型：                                       │
│     - BF16 (Bfloat16, Google Brain 格式)               │
│     - TF32 (TensorFloat32, 19-bit)                     │
│     - FP64 (双精度，科学计算)                           │
│                                                         │
│  3. FP32 输出：                                          │
│     - 所有格式都支持 FP32 累加器                         │
│     - 提高数值稳定性                                    │
│                                                         │
│  4. 稀疏化升级：                                         │
│     - 2:4 结构化稀疏 (所有格式)                         │
│     - 4:8 结构化稀疏 (部分格式)                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Ampere MMA 指令全集

```cuda
// FP16 GEMM (基础)
mma.sync.aligned.m16n8k8.row.col.f32.f16.f16.f32
    {d0, d1, d2, d3}, {a0, a1}, {b0}, {c0, d1, d2, d3};

// BF16 GEMM (AI 训练推荐)
mma.sync.aligned.m16n8k16.row.col.f32.bf16.bf16.f32
    {d0, d1, d2, d3}, {a0, a1}, {b0}, {c0, d1, d2, d3};

// TF32 GEMM (FP32 精度加速)
mma.sync.aligned.m16n8k8.row.col.f32.tf32.tf32.f32
    {d0, d1, d2, d3}, {a0, a1}, {b0}, {c0, d1, d2, d3};

// FP64 GEMM (科学计算)
mma.sync.aligned.m8n8k4.row.col.f64.f64.f64.f64
    {d0, d1}, {a0, a1}, {b0, b1}, {c0, d1};
```

### TF32 详解

```
TF32 (TensorFloat32) 格式:

┌────────────────────────────────────────┐
│  FP32 (标准单精度)                     │
│  1 符号位 │ 8 指数位 │ 23 尾数位 (32-bit)│
│  精度：~7 位十进制数字                  │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│  TF32 (TensorFloat32)                  │
│  1 符号位 │ 8 指数位 │ 10 尾数位 (19-bit)│
│  精度：~3-4 位十进制数字                │
│  范围：与 FP32 相同                     │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│  FP16 (半精度)                         │
│  1 符号位 │ 5 指数位 │ 10 尾数位 (16-bit)│
│  精度：~3 位十进制数字                  │
│  范围：6.1e-5 ~ 65504                 │
└────────────────────────────────────────┘

优势:
- 范围同 FP32 (无需重新缩放)
- 精度高于 FP16 (训练更稳定)
- 速度同 FP16 (硬件加速)
```

### Ampere 性能对比

```
A100 规格:
- SM 数量：108 个
- 每个 SM 的 TC 数量：4 个
- 每个 TC 的吞吐：1024 FMA/cycle (m16n8k8)
- 时钟频率：1410 MHz

FP16 吞吐:
108 × 4 × 1024 × 1.41 × 2 ≈ 1248 TFLOPS (稀疏)
≈ 624 TFLOPS (稠密)

TF32 吞吐:
108 × 4 × 1024 × 1.41 ≈ 624 TFLOPS (稀疏)
≈ 312 TFLOPS (稠密)

BF16 吞吐:
同 FP16: 624 TFLOPS (稠密)

FP64 吞吐:
108 × 4 × 64 × 1.41 × 2 ≈ 9.7 TFLOPS
```

## 2.5 Ada Lovelace (SM89) - 第四代 Tensor Core

### FP8 支持

```
┌─────────────────────────────────────────────────────────┐
│               Ada Lovelace SM89 新特性                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. FP8 支持 (两种格式):                                │
│     - E4M3: 4 指数位，3 尾数位 (范围小，精度高)        │
│     - E5M2: 5 指数位，2 尾数位 (范围大，精度低)        │
│                                                         │
│  2. 新增 MMA 形状：                                      │
│     - m16n8k32 (FP8 专用)                               │
│     - m32n8k16 (部分场景)                               │
│                                                         │
│  3. 稀疏化升级：                                         │
│     - 2:4 稀疏 (所有格式)                               │
│     - 4:8 稀疏 (FP8)                                    │
│                                                         │
│  4. 性能提升：                                          │
│     - FP8 吞吐：2× BF16/FP16                           │
│     - 推理场景：最高 4× RTX 3090                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### FP8 格式详解

```
FP8 E4M3 (Max Format):
┌────────────────────────────────────────┐
│  1 符号位 │ 4 指数位 │ 3 尾数位 (8-bit) │
│  范围：0 ~ 448                        │
│  精度：~1 位十进制数字                 │
│  特殊：NaN 表示为 0b01111111          │
└────────────────────────────────────────┘

FP8 E5M2 (Standard Format):
┌────────────────────────────────────────┐
│  1 符号位 │ 5 指数位 │ 2 尾数位 (8-bit) │
│  范围：6.1e-5 ~ 57344                 │
│  精度：~1 位十进制数字                 │
│  特殊：支持 Inf, NaN                   │
└────────────────────────────────────────┘

对比:
┌──────────┬──────────┬──────────┬──────────┐
│ 格式     │ 范围     │ 精度     │ 适用场景 │
├──────────┼──────────┼──────────┼──────────┤
│ FP8 E4M3 │ 小       │ 较高     │ 激活值   │
│ FP8 E5M2 │ 大       │ 较低     │ 权重     │
│ FP16     │ 中       │ 中       │ 通用     │
│ BF16     │ 大       │ 中       │ 训练     │
│ TF32     │ 大       │ 高       │ 推理     │
│ FP32     │ 大       │ 很高     │ 精度敏感 │
└──────────┴──────────┴──────────┴──────────┘
```

## 2.6 Hopper (SM90) - 第五代 Tensor Core

### GMMA (Generic MMA) 架构

```
┌─────────────────────────────────────────────────────────┐
│                   Hopper SM90 架构                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. GMMA (Generic MMA):                                 │
│     - 更大的矩阵形状                                    │
│     - m64n8k32, m64n16k32, m64n32k32                   │
│     - m128n8k32, m128n16k32, m128n32k32                │
│     - m256n8k32 (最大)                                  │
│                                                         │
│  2. 新增数据类型：                                       │
│     - FP8 (完整支持)                                    │
│     - FP64 (性能翻倍)                                   │
│     - FP128 (四精度，新支持)                           │
│                                                         │
│  3. Transformer 引擎：                                  │
│     - FP8 自动混合精度                                  │
│     - 动态缩放因子                                      │
│     - 训练时 FP8，推理时 INT8                           │
│                                                         │
│  4. 异步执行：                                          │
│     - TMA (Tensor Memory Accelerator)                  │
│     - 异步 MMA 指令                                     │
│     - 软件流水线优化                                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Hopper GMMA 指令

```cuda
// FP8 GEMM (最大吞吐)
mma.sync.aligned.m64n8k32.row.col.f32.f8.f8.f32
    {d0, d1, d2, d3, d4, d5, d6, d7},  // 8×FP32
    {a0, a1, a2, a3, a4, a5, a6, a7},  // 8×FP8
    {b0, b1, b2, b3},                   // 4×FP8
    {c0, c1, c2, c3, c4, c5, c6, c7};  // 8×FP32

// FP16 GEMM (大规模)
mma.sync.aligned.m128n16k32.row.col.f32.f16.f16.f32
    {...}, {...}, {...}, {...};

// FP64 GEMM (科学计算加速)
mma.sync.aligned.m64n8k32.row.col.f64.f64.f64.f64
    {...}, {...}, {...}, {...};

// FP128 GEMM (极高精度)
mma.sync.aligned.m32n8k32.row.col.f128.f128.f128.f128
    {...}, {...}, {...}, {...};
```

### Hopper 性能爆炸

```
H100 规格:
- SM 数量：132 个
- 每个 SM 的 TC 数量：4 个
- GMMA 吞吐：4096 FMA/cycle (m64n8k32)
- 时钟频率：1980 MHz

FP8 吞吐 (稠密):
132 × 4 × 4096 × 1.98 × 2 ≈ 8528 TFLOPS
≈ 8.5 PFLOPS

FP8 吞吐 (稀疏):
≈ 17 PFLOPS

FP16 吞吐 (稠密):
≈ 989 TFLOPS

FP64 吞吐:
≈ 34 TFLOPS (比 A100 快 3 倍)
```

## 2.8 选择合适的架构

### 应用场景推荐

```
深度学习训练:
┌──────────────────────────────────────────────┐
│  推荐：SM80 (A100) 或 SM90 (H100)           │
│  数据类型：BF16 或 FP16                      │
│  理由：数值稳定性 + 高吞吐                   │
└──────────────────────────────────────────────┘

深度学习推理:
┌──────────────────────────────────────────────┐
│  推荐：SM89 (RTX 4090) 或 SM80 (A100)       │
│  数据类型：FP8 (SM89) 或 INT8 (SM75+)        │
│  理由：低功耗 + 高吞吐                       │
└──────────────────────────────────────────────┘

科学计算:
┌──────────────────────────────────────────────┐
│  推荐：SM80 (A100) 或 SM90 (H100)           │
│  数据类型：FP64 或 FP128 (SM90)             │
│  理由：高精度需求                            │
└──────────────────────────────────────────────┘

边缘设备:
┌──────────────────────────────────────────────┐
│  推荐：SM75 (Jetson Xavier)                 │
│  数据类型：INT8 或 FP16                      │
│  理由：功耗限制                              │
└──────────────────────────────────────────────┘
```
