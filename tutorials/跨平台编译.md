# 跨平台编译指南

## 支持的 GPU

| GPU | 架构 | SM 版本 | 状态 |
|-----|------|---------|------|
| RTX 3060 (笔记本) | Ampere | 86 | ✅ 支持 |
| RTX 3070/3080/3090 | Ampere | 86 | ✅ 支持 |
| A100 | Ampere | 80 | ✅ 支持 |
| RTX 4090 | Ada Lovelace | 89 | ✅ 支持 |
| H100 | Hopper | 90 | ⚠️ 需要 SM90 |

## 快速编译

### 方法 1：CMake (推荐)

```bash
cd /mnt/d/cute-learn/tutorials
mkdir -p build && cd build
cmake ..
make -j8
```

### 方法 2：手动编译 (指定架构)

```bash
# RTX 3060 (SM86)
nvcc -std=c++17 -arch=sm_86 01_layout/01_layout_basic.cu -o 01_layout_basic

# A100 (SM80)
nvcc -std=c++17 -arch=sm_80 01_layout/01_layout_basic.cu -o 01_layout_basic

# 同时支持两种架构 (fatbin)
nvcc -std=c++17 -arch=sm_80 -gencode arch=compute_86,code=sm_86 01_layout/01_layout_basic.cu -o 01_layout_basic
```

## 编译选项详解

### CMAKE_CUDA_ARCHITECTURES

```cmake
# 仅支持 RTX 3060
set(CMAKE_CUDA_ARCHITECTURES 86)

# 仅支持 A100
set(CMAKE_CUDA_ARCHITECTURES 80)

# 同时支持 (当前配置)
set(CMAKE_CUDA_ARCHITECTURES 80 86)

# 支持所有 Ampere GPU
set(CMAKE_CUDA_ARCHITECTURES 80 86 87)

# 支持 Ampere + Ada
set(CMAKE_CUDA_ARCHITECTURES 80 86 89)
```

### 编译时间 vs 运行兼容性

| 配置 | 编译时间 | 二进制大小 | 兼容性 |
|------|----------|-----------|--------|
| `80` | 快 | 小 | A100 专用 |
| `86` | 快 | 小 | RTX 3060 专用 |
| `80 86` | 中等 | 中等 | 两者通用 ✅ |
| `80-real 86` | 慢 | 大 | 包含 PTX |

## 在不同设备上运行

### RTX 3060 笔记本

```bash

cd /mnt/d/cute-learn/tutorials
rm -rf build && mkdir build && cd build
cmake -DCMAKE_CUDA_ARCHITECTURES=86 ..
make -j8
```

## 故障排除

### 问题：编译时提示 "No CUDA support"

解决：检查 `nvidia-smi` 确认 GPU 和驱动正常

### 问题：运行时提示 "no CUDA-capable device"

解决：确认当前 GPU 支持 CUDA，检查驱动版本

### 问题：二进制文件过大

解决：只编译需要的架构
```cmake
set(CMAKE_CUDA_ARCHITECTURES 86)  # 仅笔记本
```
