# Swizzle 学习总结

## 完整学习路径回顾

### 第一步：理解概念 (01_什么是 swizzle.md)

**核心问题**：GPU 共享内存的 Bank Conflict

```
32 个线程访问同一 Bank -> 串行执行 32 次！
使用 Swizzle 分散访问 -> 并行执行 1 次！
```

**关键理解**：
- Swizzle 是一种地址变换技术
- 目的是避免 Bank Conflict
- 核心运算是 XOR

### 第二步：数学原理 (02_swizzle 数学原理.md)

**核心公式**：
```cpp
swizzled_offset = offset ^ (((offset >> SShift) & mask) << BBits);
```

**步骤分解**：
1. `offset >> SShift` - 提取高位
2. `& mask` - 限制范围
3. `<< BBits` - 对齐位置
4. `^` - 执行 XOR 翻转

**关键性质**：XOR 是可逆的！

### 第三步：参数选择 (03_参数详解.md)

**三个参数**：
| 参数 | 含义 | 计算 | 推荐值 |
|------|------|------|--------|
| BBits | Bank 大小对数 | `bank_size = 2^BBits` | 2 (float) |
| NumBanks | Bank 数量对数 | `num_banks = 2^NumBanks` | 5 (32 银行) |
| SShift | 位移量 | 必须 `>= BBits` | `BBits + 1` |

**推荐配置**：
- float32: `Swizzle<2, 5, 3>`
- float16: `Swizzle<3, 5, 4>`
- int8: `Swizzle<4, 5, 5>`

## 核心知识点总结

### 1. Swizzle 的本质

```
Swizzle = XOR 地址变换
目的 = 分散 Bank 访问
效果 = 避免 Bank Conflict
```

### 2. 参数选择口诀

```
BBits 看元素大小
NumBanks 看硬件
SShift 大于 BBits 就可以
```

### 3. 常用配置

```cpp
// 最常用：float 类型
Swizzle<2, 5, 3>

// GEMM 共享内存
Swizzle<2, 4, 3>

// TMA 拷贝 (Hopper)
Swizzle<2, 5, 3>
```

### 4. 使用模式

```cpp
// 1. 直接函数调用
int swizzled = Swizzle<2, 4, 3>::apply(offset);

// 2. 与 Layout 组合
auto layout = make_layout(shape);
auto swizzle_fn = Swizzle<2, 4, 3>{};
auto swizzled_layout = composition(swizzle_fn, layout);

// 3. 在 Kernel 中使用
__shared__ float shared[256];
int idx = Swizzle<2, 4, 3>::apply(threadIdx.x);
shared[idx] = value;
```

## 常见问题解答

### Q1: 为什么我看到的 Swizzle 没有变化？
A: 可能 SShift 太大，前几个元素右移后为 0，XOR 值为 0。尝试更大的 offset。

### Q2: SShift 可以小于 BBits 吗？
A: 不可以！这会导致不可逆变换。

### Q3: 如何选择合适的参数？
A: 从推荐配置开始：`Swizzle<2, 5, 3>`（float），然后根据需要调整。

### Q4: Swizzle 真的能提升 27 倍性能？
A: 在特定场景下（如矩阵转置的 Bank Conflict 严重时）可以。实际提升取决于访问模式。
