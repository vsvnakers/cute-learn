# Swizzle 可视化演示

## 7.1 一维地址变换可视化

### Swizzle<2, 4, 3> 的完整映射表

```
输入偏移 (0-31) 的二进制表示和 Swizzle 结果:

偏移  二进制   右移 3 位   Mask(15)   XOR 值    结果    二进制
--------------------------------------------------------------
 0    000000      0         0         0       0      000000
 1    000001      0         0         0       1      000001
 2    000010      0         0         0       2      000010
 3    000011      0         0         0       3      000011
 4    000100      0         0         0       4      000100
 5    000101      0         0         0       5      000101
 6    000110      0         0         0       6      000110
 7    000111      0         0         0       7      000111
 8    001000      1         1         4      12      001100  ← 翻转了 bit 2
 9    001001      1         1         4      13      001101
10    001010      1         1         4      14      001110
11    001011      1         1         4      15      001111
12    001100      1         1         4       8      001000  ← 翻转了 bit 2
13    001101      1         1         4       9      001001
14    001110      1         1         4      10      001010
15    001111      1         1         4      11      001011
16    010000      2         2         8      24      011000  ← 翻转了 bit 3
...
```

### 关键观察

1. **前 8 个元素（0-7）不变**：因为右移 3 位后为 0
2. **8-15 元素翻转 bit 2**：XOR 值为 4 (100)
3. **16-23 元素翻转 bit 3**：XOR 值为 8 (1000)

### 变换规律

```
观察：
- Row 0-3 和 Row 4-7 交换了位置（在每列的前 4 个元素内）
- 这是因为 XOR 翻转了第 2 位（值为 4）

用二进制看 Row 索引:
Row 0 = 000 -> 000 = Row 0  (XOR 0)
Row 4 = 100 -> 100 = Row 4  (XOR 0, 因为高位为 0)

等等...
```

## 7.3 Bank 分布可视化

### 无 Swizzle 的 Bank 分布问题

```
假设 4 个 Bank，每个 Bank 存储 4 个连续元素：

Bank 0: 地址 0-3, 16-19, 32-35, ...
Bank 1: 地址 4-7, 20-23, 36-39, ...
Bank 2: 地址 8-11, 24-27, 40-43, ...
Bank 3: 地址 12-15, 28-31, 44-47, ...

问题场景：访问一列的所有元素
R0C0: 地址 0 -> Bank 0
R1C0: 地址 16 -> Bank 0  ← 冲突！
R2C0: 地址 32 -> Bank 0  ← 冲突！
R3C0: 地址 48 -> Bank 0  ← 冲突！

结果：4 次访问必须串行执行！
```

### 使用 Swizzle 后的 Bank 分布

```
应用 Swizzle<2, 4, 3> 后:

R0C0: 地址 0 -> Swizzled 0 -> Bank 0
R1C0: 地址 16 -> Swizzled 24 -> Bank 2  ← 分散了！
R2C0: 地址 32 -> Swizzled 48 -> Bank 3  ← 分散了！
R3C0: 地址 48 -> Swizzled 32 -> Bank 2  ← 分散了！

结果：访问分散到不同 Bank，可以并行执行！
```

## 7.4 交互式可视化（代码）

### 打印任意 Swizzle 配置的效果

```cpp
void visualize_swizzle(int BBits, int NumBanks, int SShift) {
    std::cout << "Swizzle<" << BBits << ", " << NumBanks << ", " << SShift << ">" << std::endl;
    std::cout << "  Bank Size: " << (1 << BBits) << std::endl;
    std::cout << "  Num Banks: " << (1 << NumBanks) << std::endl;
    std::cout << std::endl;

    std::cout << "偏移映射 (前 32 个):" << std::endl;
    for (int i = 0; i < 32; i++) {
        int s = Swizzle<BBits, NumBanks, SShift>::apply(i);
        int orig_bank = (i >> BBits) & ((1 << NumBanks) - 1);
        int swizz_bank = (s >> BBits) & ((1 << NumBanks) - 1);

        std::cout << std::setw(2) << i << " -> " << std::setw(2) << s
                  << " [B" << orig_bank << " -> B" << swizz_bank << "]";

        if ((i + 1) % 4 == 0) std::cout << std::endl;
    }
}
```

## 7.5 动画式理解

### Swizzle 如何"交换"数据

想象一个 8 元素的数组：

```
初始状态: [0, 1, 2, 3, 4, 5, 6, 7]
          |--------| |--------|
          第一组    第二组

Swizzle<2,3,3> 的效果:
- 前 4 个元素 (0-3) 不动
- 后 4 个元素 (4-7) 内部交换

步骤:
1. offset=4: 4>>3=0, XOR 值=0, 结果=4  不变
2. offset=5: 5>>3=0, XOR 值=0, 结果=5  不变
...

等等，好像不对？让我重新计算...

实际上 Swizzle<2,3,3>:
- BBits=2, NumBanks=3, SShift=3
- 对于 offset >= 8，右移 3 位才非零

所以:
- 0-7: 不变 (右移 3 位 = 0)
- 8-15: XOR 4 (右移 3 位 = 1, mask=7, 1<<2=4)
```
