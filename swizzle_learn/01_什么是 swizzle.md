# 第一课：什么是 Swizzle？

## 1.1 从实际问题出发

想象一下，你有一个 GPU 共享内存（Shared Memory），它被分成了 **32 个 Bank**（银行）。

```
Bank 0   Bank 1   Bank 2   ...  Bank 31
  |        |        |            |
地址 0    地址 1   地址 2   ...  地址 31
地址 32   地址 33  地址 34  ...  地址 63
地址 64   地址 65  地址 66  ...  地址 95
...
```

**规则**：每个 Bank 在同一时间只能服务一个访问请求。

### 问题场景

假设你有 32 个线程，每个线程想访问共享内存的一个元素：

```cuda
// 糟糕的访问模式
int col = 0;  // 所有线程都访问第 0 列
int row = threadIdx.x;
float value = shared_mem[row * 32 + col];  // 所有地址都在 Bank 0！
```

会发生什么？
- 线程 0 访问地址 0 (Bank 0)
- 线程 1 访问地址 32 (Bank 0)
- 线程 2 访问地址 64 (Bank 0)
- ...
- 线程 31 访问地址 992 (Bank 0)

**结果**：32 次访问全部落在 Bank 0，必须**串行执行**32 次！性能损失 32 倍！

这就是 **Bank Conflict**。

## 1.2 Swizzle 是什么？

**Swizzle 的本质**：一种地址变换技术，通过 XOR 运算重新排列内存地址。

**目的**：让原本可能落在同一 Bank 的地址，分散到不同的 Bank。

### 直观理解

```
原始地址：0,  1,  2,  3,  4,  5,  6,  7
           |  |  |  |  |  |  |  |
           v  v  v  v  v  v  v  v
Bank:      0  1  2  3  0  1  2  3   (假设有 4 个 Bank)

问题：访问列 0,4,8,12 都会落在 Bank 0！

Swizzle 后：
原始地址：0,  1,  2,  3,  4,  5,  6,  7
           |  |  |  |  |  |  |  |
           v  v  v  v  v  v  v  v
Swizzled:  0,  8,  4,  12, 6,  14, 2,  10
           |   |   |   |   |   |   |   |
           v   v   v   v   v   v   v   v
Bank:      0   2   1   3   2   3   1   0  (分散开了！)
```

## 1.3 Swizzle 的名字来源

"Swizzle" 这个词来自英语，意思是"旋转"、"变换"。

在图形学中，"swizzle" 指重新排列向量的分量，比如：
```cpp
vec4.rgba  ->  vec4.bgra  (这是一种 swizzle 操作)
```

在 CUDA 中，Swizzle 指**地址的重新排列**。

## 1.4 为什么需要学习 Swizzle？

1. **GEMM 优化**：矩阵乘法中共享内存访问必须用 Swizzle
2. **Attention 优化**：Flash Attention 等算法依赖 Swizzle
3. **面试考点**：GPU 优化工程师必考
4. **性能关键**：正确使用可提升 2-32 倍性能

## 1.5 小结

| 概念 | 解释 |
|------|------|
| Bank | 共享内存的分区，每个 Bank 独立工作 |
| Bank Conflict | 多个线程访问同一 Bank，导致串行化 |
| Swizzle | 地址变换技术，分散访问到不同 Bank |
| 核心思想 | XOR 运算重新排列地址 |
