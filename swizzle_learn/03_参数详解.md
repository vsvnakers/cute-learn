# 第三课：Swizzle 参数详解

## 3.1 CUTE Swizzle 的三个参数

CUTE 中的 Swizzle 定义为：

```cpp
Swizzle<BBits, NumBanks, SShift>
```

三个参数都是**模板参数**（编译时常量）。

### 参数含义速查表

| 参数 | 名称 | 含义 | 计算 |
|------|------|------|------|
| `BBits` | Bank Bits | Bank 大小的对数 | `bank_size = 1 << BBits` |
| `NumBanks` | Num Banks | Bank 数量的对数 | `num_banks = 1 << NumBanks` |
| `SShift` | Shift Amount | XOR 位移量 | 决定哪些位参与 XOR |

## 3.2 BBits - Bank 大小

`BBits` 决定了**每个 Bank 能容纳的元素数量**。

```
bank_size = 2^BBits
```

### 常见取值

| BBits | Bank Size | 含义 |
|-------|-----------|------|
| 2 | 4 | 每个 Bank 4 个元素 |
| 3 | 8 | 每个 Bank 8 个元素 |
| 4 | 16 | 每个 Bank 16 个元素 |
| 5 | 32 | 每个 Bank 32 个元素 |

### 如何选择？

对于 `float`（4 字节）：
- 如果希望每个 Bank 存储 16 字节（4 个 float），选 `BBits = 2`
- 如果希望每个 Bank 存储 32 字节（8 个 float），选 `BBits = 3`

**经验法则**：`BBits = 2` 是最常用的选择。

## 3.3 NumBanks - Bank 数量

`NumBanks` 决定了**Bank 的数量**。

```
num_banks = 2^NumBanks
```

### 常见取值

| NumBanks | Num Banks | 含义 |
|----------|-----------|------|
| 4 | 16 | 16 个 Bank |
| 5 | 32 | 32 个 Bank（标准配置）|
| 6 | 64 | 64 个 Bank |

### GPU 硬件限制

- Ampere (A100): 32 个 Bank
- Hopper (H100): 32 个 Bank（但支持 TMA）

**经验法则**：`NumBanks = 5`（即 32 个 Bank）是标准配置。

## 3.4 SShift - 位移量

`SShift` 是最关键的参数，决定了**哪些位参与 XOR 运算**。

### 约束条件

```
SShift >= BBits
```

**为什么？** 如果 `SShift < BBits`，XOR 会破坏低位信息，导致不可逆！

### 工作原理

```
offset = xxxxxxxxxxxx (二进制)
              ^^^^^^  低位 BBits（Bank 内偏移）
         ^^^^^^       高位部分（决定是否翻转）

SShift = 3 意味着：
- 取 offset >> 3 的结果
- 用这个结果决定是否翻转低位
```

### SShift 大小的影响

```
Swizzle<2, 4, 3>: 每 8 个元素"翻转"一次
Swizzle<2, 4, 4>: 每 16 个元素"翻转"一次
Swizzle<2, 4, 5>: 每 32 个元素"翻转"一次
```

**经验法则**：`SShift = BBits + NumBanks` 或更大。

## 3.5 参数组合示例

### 标准配置（推荐）

```cpp
// 最常用：用于 16x16 矩阵的共享内存
Swizzle<2, 4, 3>  // BBits=2, NumBanks=4, SShift=3

// 用于 32x32 矩阵
Swizzle<3, 5, 4>  // BBits=3, NumBanks=5, SShift=4

// 用于 TMA 拷贝
Swizzle<2, 5, 3>  // BBits=2, NumBanks=5, SShift=3
```

### 参数对比表

| 配置 | Bank Size | Num Banks | SShift | 适用场景 |
|------|-----------|-----------|--------|----------|
| `<2,4,3>` | 4 | 16 | 3 | 小型矩阵 |
| `<2,5,3>` | 4 | 32 | 3 | 标准配置 |
| `<3,5,4>` | 8 | 32 | 4 | 大型矩阵 |
| `<4,5,5>` | 16 | 32 | 5 | 特殊场景 |

## 3.6 参数选择的经验法则

### 法则 1：先确定 BBits

根据元素大小：
- `float` (4B): `BBits = 2`（每个 Bank 4 元素 = 16 字节）
- `half` (2B): `BBits = 3`（每个 Bank 8 元素 = 16 字节）

### 法则 2：NumBanks 通常固定

- 大部分情况：`NumBanks = 5`（32 个 Bank）
- 特殊情况（16 Bank 硬件）：`NumBanks = 4`

### 法则 3：SShift 满足约束

```
SShift >= BBits + log2(访问模式的周期)
```

**安全选择**：`SShift = BBits + NumBanks`

## 3.7 实际应用中的参数

### GEMM 共享内存 Layout

```cpp
// 16x16 tile 的共享内存（每线程 8 元素）
using SharedLayout = Swizzle<2, 4, 3>;
```

### TMA 拷贝（Hopper）

```cpp
// TMA 要求的 Swizzle 格式
using TMALayout = Swizzle<2, 5, 3>;
```

### Warp 级别的矩阵

```cpp
// 32x8 矩阵（warp 矩阵）
using WarpLayout = Swizzle<3, 5, 4>;
```

## 3.8 参数验证工具

写一个简单的程序验证参数：

```cpp
template<int BBits, int NumBanks, int SShift>
void verify_swizzle() {
    constexpr int bank_size = 1 << BBits;
    constexpr int num_banks = 1 << NumBanks;

    printf("Swizzle<%d,%d,%d>:\n", BBits, NumBanks, SShift);
    printf("  Bank Size: %d\n", bank_size);
    printf("  Num Banks: %d\n", num_banks);
    printf("  Constraint: SShift(%d) >= BBits(%d)? %s\n",
           SShift, BBits, SShift >= BBits ? "OK" : "FAIL");
}
```

## 3.9 常见错误

### 错误 1：SShift 太小

```cpp
// 错误！SShift < BBits
Swizzle<3, 4, 2>  // SShift=2 < BBits=3，不可逆！
```

### 错误 2：NumBanks 与硬件不匹配

```cpp
// 在 32-Bank GPU 上使用 16-Bank 配置
Swizzle<2, 4, 3>  // 可能无法充分利用所有 Bank
```

### 错误 3：不考虑元素大小

```cpp
// 对 half 类型使用 float 的 BBits
// half 应该用更大的 BBits
Swizzle<2, 5, 3>  // 对 half 可能不是最优
```

## 3.10 小结

```
选择参数的步骤：
1. 确定元素大小 -> 选择 BBits
2. 确认硬件 -> 选择 NumBanks
3. 满足约束 -> 选择 SShift (>= BBits)
4. 验证效果 -> 运行测试代码
```
